Diabeto

May 2016

Created a logbook-style Android application for Type 1 diabetics to record blood glucose levels, carb intake, and insulin dosages. Utilized an SQLite database for local storage, fragments and a nav drawer for the UI, the androidplot library for graphing, background services for recurring notifications, and a custom decision tree implementation for blood glucose prediction.

Startup screen is shown on launch during a quick check for new or updated bus stops.
After startup a Google Map view is shown. Users can center on their location and select nearby bus stops.
The slide-out navigation drawer is the app's primary navigation element.
Users can also find stops by searching using CUMTD's auto-complete API.
Up to four of the users most recent stop selections are cached and shown when search is selected.
Once a stop is selected, the 'stopwatch' view shows upcoming departures. If the stop has multiple pickup locations, a sliding tab layout allows users to easily switch between them. The stop can be favorited using the star in the top-right corner.
Favorite stops can be viewed in this fragment. When calculation is complete, the loading indicators are replaced with the distance (in miles) to each stop.
