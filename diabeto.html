Diabeto

April 2016 - May 2016

Created a logbook-style Android application for Type 1 diabetics to record blood glucose levels, carb intake, and insulin dosages. Utilized an SQLite database for local storage, fragments and a nav drawer for the UI, the androidplot library for graphing, background services for recurring notifications, and a custom decision tree implementation for blood glucose prediction.

The slide-out navigation drawer allows users to switch between fragments.
A completed log entry can be edited or deleted using the nav-bar buttons in the top right.
The 'status' fragment shows current statistics calculated using today's entries, and user preferences for active insulin time, basal rate, and carbohydrate ratio.
This log input form provides an easy way for users to record blood glucose, bolus, and/or carbohydrate.
The 'logs' fragment shows an expandable list view organized by date.

For recent dates, simple identifiers like 'today' or 'Sunday' are used instead of the full date.
When the device is rotated into landscape a graph plotting today's log entries vs. time fills the screen.
The log input form defaults to current date/time, but the user can change either value if necessary.
Blood glucose readings are color coded acccording to the user's preferences for hypo/hyper-glycemic values.

New entries can be added using the FloatingActionButton in the bottom-right corner.
